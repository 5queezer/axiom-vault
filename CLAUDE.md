# AxiomVault

Cross-platform encrypted vault with client-side encryption. Files are encrypted locally before touching any cloud service.

## Tech Stack

- **Language:** Rust 2021 edition, MSRV 1.85.0
- **Crypto:** XChaCha20-Poly1305 (AEAD), Argon2id (KDF), Blake2b (hashing)
- **Desktop:** Tauri 2.0 (legacy), GTK4/libadwaita (Linux native)
- **Mobile:** C-ABI FFI via `core/ffi` + cbindgen → Swift (iOS/macOS), Kotlin (Android)
- **Filesystem:** FUSE via `fuser` crate for virtual mount
- **Storage:** Google Drive (OAuth2), extensible via storage provider trait
- **DB:** SQLite via `rusqlite` (bundled)

## Commands

```bash
# Format check
cargo fmt --all -- --check

# Lint (warnings are errors)
cargo clippy --workspace --all-targets --all-features -- -D warnings

# Build
cargo build --workspace --verbose

# Test
cargo test --workspace --verbose

# Doc build (warnings are errors)
RUSTDOCFLAGS="-D warnings" cargo doc --workspace --no-deps

# FFI header generation
cargo build -p axiom-ffi --release
# Produces: target/include/axiom_ffi.h

# macOS: exclude Linux GTK client from build/test
cargo build --workspace --exclude axiomvault-linux --verbose
cargo test --workspace --exclude axiomvault-linux --verbose
```

## Architecture

```text
core/
  crypto/     # Encryption, key derivation, recovery keys (bip39)
  vault/      # Vault engine, config, tree index
  storage/    # Storage provider trait + Google Drive implementation
  sync/       # Sync engine, conflict resolution
  fuse/       # FUSE virtual filesystem mount
  ffi/        # C-ABI bindings for mobile (cbindgen)
  common/     # Shared types across crates
  app/        # Application core, orchestrates other crates
clients/
  apple/      # Unified iOS + macOS (SwiftUI, XcodeGen)
  android/    # Android (Kotlin Compose)
  linux/      # Linux native desktop (GTK4/libadwaita)
  desktop/    # Legacy Tauri desktop GUI
tools/
  cli/        # Command-line interface (clap)
```

## Coding Conventions

- Use `thiserror` for library crate errors, `anyhow` only in binaries and tests
- `zeroize` all sensitive data: keys, plaintext buffers, passwords. Derive `Zeroize`/`ZeroizeOnDrop` on types holding secrets
- Every `unsafe` block requires a `// SAFETY:` comment explaining the invariant. Enforced by pre-commit hook
- Async runtime is `tokio` with `async-trait` for trait methods
- Errors propagate with `?`; avoid `.unwrap()` in library code

## Gotchas

- **System deps for Linux build:** `libfuse3-dev`, `libgtk-4-dev`, `libadwaita-1-dev`, plus Tauri deps (`libgtk-3-dev`, `libwebkit2gtk-4.1-dev`, `libappindicator3-dev`, `librsvg2-dev`, `patchelf`)
- **SQLite is bundled:** `rusqlite` uses `features = ["bundled"]` — no system SQLite needed, but increases compile time
- **FFI header location:** `target/include/axiom_ffi.h` is generated by cbindgen during `axiom-ffi` release build. Regenerate after changing FFI-exposed types
- **macOS CI excludes `axiomvault-linux`:** The Linux GTK client depends on GTK4/libadwaita which aren't available on macOS. Always use `--exclude axiomvault-linux` when building on macOS
- **Recovery keys use BIP-39:** Mnemonic word encoding for vault recovery keys via `bip39` crate

## CI & Automation

- CI runs on push to `master`, `develop`, and `claude/**` branches
- Never commit directly to `master` — use feature branches and PRs
- Pre-commit hooks (install: `pre-commit install`):
  - `gitleaks` — scans for leaked secrets
  - `cargo fmt --check` — format enforcement
  - `cargo clippy` — lint with `-D warnings`
  - Unsafe block SAFETY comment check
  - Build with `-D warnings`
- CI matrix tests on Ubuntu + macOS, Rust stable + MSRV (1.85.0)
- Security audit runs `cargo audit` on every push
